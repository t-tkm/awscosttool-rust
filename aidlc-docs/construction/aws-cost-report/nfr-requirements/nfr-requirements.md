# NFR Requirements - aws-cost-report

## パフォーマンス
- **起動時間**: 特別な要件なし（CLI ツールのため数秒以内で十分）
- **API レスポンス**: AWS Cost Explorer / STS の応答待ちが支配的。タイムアウトはデフォルト SDK 設定に従う
- **メモリ使用量**: 特別な要件なし（サービス数は高々数百件）

## 可用性・信頼性
- **実行モデル**: 単発実行（デーモンではない）
- **リトライ**: AWS SDK のデフォルトリトライ設定を使用（指数バックオフ）
- **エラー時の挙動**: エラーメッセージを stderr に出力し、非ゼロ終了コードで終了

## セキュリティ
- **認証**: AWS 認証情報は環境変数・プロファイル・IAM ロール等の標準 SDK チェーンに委譲
- **最小権限**: 実行に必要な IAM 権限は `ce:GetCostAndUsage` と `sts:GetCallerIdentity` のみ
- **シークレット管理**: ハードコードされた認証情報なし

## 保守性
- **コード品質**: `cargo clippy` 警告ゼロ、`cargo fmt` 適用
- **テスト**: ビジネスロジック（日付計算・コスト集計・フォーマット）のユニットテストを実装
- **エラー型**: `anyhow` または `thiserror` を使用した明示的なエラー型

## ログ出力
- **フレームワーク**: `env_logger` または `tracing` + `tracing-subscriber`
- **レベル**: INFO（通常実行）、DEBUG（除外サービスの詳細）、ERROR（API 失敗）
- **制御**: `RUST_LOG` 環境変数で制御可能
